# 系统技术迭代路线

## 短期（0 外部依赖或 pip install 一个库）

---

### 1. 技术形态信号 (金叉/死叉/背离/突破)

**方案**: `pip install pandas-ta`，直接调 `df.ta.crossover()` / `df.ta.rsi()` 等生成二进制信号列。

**解决的问题**:
- 当前系统用原始因子值做匹配，但两段走势的 MA20 从 1300 涨到 1400 vs 从 800 涨到 900，形态相似但绝对值差异大，Pearson/DTW 都受影响
- 换成"MA5 上穿 MA20 = 1"这种二进制信号 → 匹配的是事件节奏而非价格水平 → 跨时间可比性大幅提升
- KDJ/RSI 的超买超卖区域也是天然的二值信号

**提升**: 匹配命中率预计 +3~5 个百分点，尤其利好跨长时间段（>3 年）的搜索。

---

### 2. 集成预测 (多时间窗口投票)

**方案**: 同时跑预测天数 = 3/5/10 三个版本，按多数表决出最终方向。

**解决的问题**:
- 当前单一预测窗口（如 5 天），某一次匹配的历史 5 天后走势是涨的，但 3 天后先跌了 10% → 方向判断"对"但无法指导实际进出
- 三窗投票 → 3 天看涨 + 5 天看涨 + 10 天看涨 → 强信号；3 天看涨 + 10 天看跌 → 弱信号，过滤掉

**提升**: 减少单一窗口的噪音干扰，强信号命中率可比单一窗口高 5~8 个百分点。

---

### 3. 择时过滤 (波动率门控)

**方案**: 当前 `vol20d` 或 `vol60d` 因子已存在。加一个规则：波动率 > 历史 80% 分位数 → 当日不出信号（市场太乱，匹配失效）。

**解决的问题**:
- 突发新闻、财报日、政策冲击 → 价格剧烈波动 → 任何历史形态匹配都失效
- 当前系统在这些日子预测，命中率接近随机（50%），拖累全局统计

**提升**: 剔除 20% 的高波动日后，剩余信号命中率可提升 3~5 个百分点。

---

### 4. 跨股票批量回测

**方案**: 写一个循环，传入多个股票代码，分别跑自动调参，汇总最优参数，看哪些参数是跨股票通用的。

**解决的问题**:
- 当前只验证了茅台、五粮液两只票，策略的通用性完全未知
- 可能茅台适合窗口 30 天、五粮液适合 10 天 → 背后是流动性/波动率/机构持仓比在起作用 → 可挖掘元特征

**提升**: 验证策略可迁移性；积累一批"适合此策略"的股票池。

## 中期（pip install 一个库 + 少量代码）

---

### 5. 趋势分层回测 (牛市/熊市/震荡市)

**方案**: 用 MA60 / MA250 的相对位置自动贴标签：
- `close > MA250` 且 MA60 向上 → 牛市
- `close < MA250` 且 MA60 向下 → 熊市
- 其余 → 震荡

分别统计三种市况下的信号段命中率。

**解决的问题**:
- 当前全周期混在一起统计，但形态匹配在趋势市和震荡市的行为完全不同
- 趋势市：惯性大，匹配更容易命中（但可能是假信号，因为"随便买都涨"）
- 震荡市：形态来回反转，匹配准确率才是策略真实水平的检验

**提升**: 能清楚回答"这策略什么时候能用、什么时候不能用"，避免在熊市里盲目信任一个牛市中校准的策略。

---

### 6. LightGBM 自动权重学习

**方案**: `pip install lightgbm`。训练数据：每次回测中，各因子对最终预测的"贡献"作为标签，用 LightGBM 学出最优权重向量。

**解决的问题**:
- 当前人工调滑块靠直觉（close 给 50%，rsi14 给 25%...），没有数据支撑
- 不同股票、不同时间段的因子重要性完全不同，一套权重无法通用

**提升**: 自动发现哪些因子在什么条件下重要 → 调参搜索从 240 种组合扩展到"因子权重 × 240"，搜索更充分 → 命中率预计 +5%。

---

### 7. 贝叶斯优化替代网格搜索

**方案**: `pip install optuna`。替代当前的 240 种穷举搜索。Optuna 用 TPE（Tree-structured Parzen Estimator）算法，几十次试算就能找到近似最优。

**解决的问题**:
- 当前网格搜索 240 种固定组合，很多组合是明显差的（如窗口 5 + 阈值 0.7），浪费算力
- 如果未来参数维度增加（加波动率门控阈值、加因子权重），网格搜索会指数爆炸

**提升**: 搜索效率 10-50x，支持未来增加更多参数维度。

## 长期（深度学习，1-2 周开发）

---

### 8. Autoencoder 因子压缩

**方案**: 用 PyTorch/Keras 搭一个 3 层 Autoencoder，输入 65 维因子，压缩到 8-16 维嵌入向量，在嵌入空间做形态匹配。

**解决的问题**:
- 65 个因子很多高度相关（rsi6 和 rsi14 相关 0.9+），信息冗余导致多因子评分被重复计算
- 压缩到嵌入空间 → 每个维度独立 → 匹配更纯粹

**提升**: 理论上能捕捉当前 Pearson/DTW 无法发现的非线性形态关系。

---

### 9. Transformer 注意力匹配

**方案**: 不用逐窗口滑动比较，直接用 Transformer 的 self-attention 机制计算当前模板与所有历史窗口的"注意力分数"，一次前向传播完成所有匹配。

**解决的问题**:
- 当前逐窗口比较是 O(n²) 复杂度，Transformer 可以并行化
- Self-attention 能捕捉"哪些历史片段和当前最相关"，比 Pearson/DTW 的预定义度量更灵活

**提升**: 实现上的终极形态，但训练需要大量历史数据，且模型解释性差。

## 量化策略审查 (2026-06-29)

### 当前系统已验证的能力

| 能力 | 证据 |
|------|------|
| 方向预测命中率 > 50% | 长周期回测 (2020-2026) 稳定找到 56-62% 配置 |
| 因子重要性自动学习 | Optuna 自动淘汰无用因子 (close 从 78%→1-6%, 取决于因子组合) |
| 时间鲁棒性 | CV 3 窗口标准差 ±3-4%, 非单段碰巧 |
| 统计样本充足 | 验证集 45-108 段, Wilson 下界可靠 |
| 跨算法一致性 | Pearson/DTW/Pearson+DTW 均收敛到短预测 (3天) 最优 |

### 设计中存在的量化问题

**1. 单股票过拟合风险**
所有测试仅针对 600519 (贵州茅台)。茅台是白酒龙头、高机构持仓、低波动——规律未必适用于小盘股、周期股。最优参数 (窗口30/预测3/TopK=1) 可能是茅台的个性化配置。

**2. 多轮搜索的多重检验问题**
四轮不同因子组合 × 每轮 400 次 Optuna 试算 × 多次导出 CSV = 累积数千次假设检验。没有 Bonferroni 校正或 hold-out 最终验证，存在"搜到为止"的选择偏差。

**3. 方向命中率 ≠ 策略盈利**
55-62% 命中率在 3 天周期上方向正确，但：
- 预测涨+2%、实际涨+0.1% 算命中但不够覆盖交易成本
- 40% 的失败交易中，平均亏损幅度未知
- "命中"只在方向，不在盈亏比

**4. 历史匹配假设的局限性**
"过去形态相似 → 未来走势相似" 依赖市场结构性稳定。A 股在 2020-2026 经历了注册制改革、外资开放、量化交易爆发——结构性变化可能削弱历史类比的有效性。

**5. 相邻窗口信号冗余**
窗口 30 天 + 每天滑动 → 连续多天的 30 天模板重叠 97%。信号段去重解决了"同质预测重复计数"，但没解决"同质预测是否应该合并为一次决策"——如果连续 3 天出同一方向信号，应该加重仓位还是保持原样？

### 需要穿透验证的方向

| 方向 | 方法 | 预计结论 |
|------|------|----------|
| **盈亏比分析** | 改造 hit/miss 统计: 命中时平均收益 vs 失败时平均亏损 | 如果命中+1.5% vs 失败-2%, 55% 命中率仍亏钱 |
| **扣费后净收益** | 每股 0.1% 交易成本 + 印花税 0.05%, 3 天净回报是否为正 | 大概率会吃掉大部分信号利润 |
| **多股票测试** | 选 5-10 只不同市值/行业 A 股, 跑相同参数, 看命中率跨股票分布 | 如果茅台 60% 其他 45% → 单股票过拟合实锤 |
| **卖出信号精度** | 当前只看涨方向, 跌方向命中率是否同样有效 | 多数策略做空精度低于做多 |
| **持仓期收益分布** | 不做方向判断, 只看历史相似片段的后续绝对收益均值是否 > 0 | 排除方向投票的影响, 直接看形态匹配的质量 |
