# 股票多因子分析系统 — 技术文档

## 1. 项目概述

一个基于 Python 的 A 股多因子分析与历史形态匹配系统。通过免费数据源拉取历史日线和财报数据，计算 65+ 个量化因子，提供交互式 Dashboard 进行可视化分析，支持三种形态匹配算法 (Pearson / DTW / Pearson+DTW)，逐日回测验证、信号段去重统计、自动参数调优，以及命中/未命中的时间分布分析。

**技术栈**: Python 3.9 + Streamlit + Plotly + AkShare + Pandas + NumPy + SciPy + Numba

**在线地址**: https://htma2024-stock-factors-app-pyb4mo.streamlit.app

**仓库**: https://github.com/HTMA2024/stock-factors

## 3. 数据架构

### 3.1 数据源

| 数据层 | 来源 | API | 频率 | 备注 |
|--------|------|-----|------|------|
| 日线行情 | 新浪财经 | `stock_zh_a_daily` | 日频 | OHLCV + 换手率 + 流通股本 |
| 估值数据 | 百度股市通 | `stock_zh_valuation_baidu` | 双周频 → 日频 | PE(TTM)/PE(静)/PB/市现率/总市值 |
| 财务报表 | 同花顺 | `stock_financial_abstract_ths` | 季度 → 日频 | ROE/毛利率/增长率/资产负债率等 |
| 指数行情 | 新浪财经 | `stock_zh_index_daily` | 日频 | 用于计算 Beta |

### 3.2 价格修正机制

新浪前复权价格因近期除权除息滞后产生偏差。系统用独立数据源交叉修正：

```
真实股价 = PB(百度) × BPS(同花顺财报)
```

- PB 来自百度双周频，线性插值为日频
- BPS 来自同花顺季报，前向填充到日频
- PE/PB/PCF 在 factor_engine 中前向填充为日频，确保形态匹配有全量搜索深度

### 3.3 数据缓存

- API 数据缓存到本地 `data/` 目录，按日期范围校验完整性，不完整自动重拉
- Streamlit `@st.cache_data` 缓存因子计算和 Pearson 相关矩阵 (TTL 1小时)
- 形态匹配相关系数矩阵独立缓存，权重调整不触发重算

## 6. 历史形态匹配

### 6.1 策略原理

**历史类比法 (Historical Analogy / Pattern Matching)**：当前一段因子曲线作为"模板"，在历史中搜索形态最相似的窗口（仅搜索模板开始前的数据，排除自身和相邻窗口），观察相似窗口之后的走势作为预测参考。

### 6.2 三种匹配算法

| 算法 | 原理 | 速度 | 加速技术 | 适用 |
|------|------|------|----------|------|
| **Pearson** | 线性相关系数 | 最快 | NumPy 矩阵乘法 (BLAS) | 节奏一致的线性趋势 |
| **DTW** | 动态时间规整, ±2天伸缩 | 中等 | Numba JIT + LB_Keogh 下界剪枝 | 同趋势但涨跌快慢不同 |
| **Pearson+DTW** | Pearson 矩阵初筛 (r≥0.3) → DTW 精选 | 中等 | 矩阵初筛 + LB_Keogh | 兼顾速度与形状灵活性 |

**DTW 加速细节**:
- **Numba JIT**: `@njit` 编译为机器码, 50-100x
- **LB_Keogh 剪枝**: O(n) 下界包络线预判, 剪掉 60%+ 无意义比较, 零精度损失
- **z-score 标准化**: DTW 计算前先标准化, 确保只比较形状不比较绝对数值
- **搜索范围约束**: 只搜索模板开始前的历史窗口, 排除自身匹配

### 6.3 多因子权重

```
综合得分 = Σ(因子_i 的相似度 × 权重_i)
```
- 权重可独立调滑块, 总和其他因子自动归一化为 1
- 6 种快捷预设一键填充因子组合

### 6.4 参数配置

| 参数 | 说明 | 选项 |
|------|------|------|
| 匹配窗口 | 模板长度 (交易日) | 5 / 10 / 20 / 30 / 60 |
| 显示前 N 名 | 展示的匹配数量 | 1 ~ 20 |
| 预测展望 | 后续走势显示天数 | 0 ~ 60 |
| 模板截止日 | 模板窗口结束日期 | 任意历史日 |
| 匹配算法 | Pearson / DTW / Pearson+DTW | 三选一 |
| 因子权重 | 各因子权重滑块 | 自动归一化 |
| 快捷预设 | 一键填因子组合 | 6 种 |

### 6.5 可视化设计

- **当前模板图**：所有选中因子叠加, X=相对天数 (0=截止日), 悬停显示实际日期
- **排名分组**：每个排名一行, 每个因子一张小图并排展示
- **单匹配图**: 蓝色实线=历史, 蓝色虚线=预测, 红色实线=当前, 灰线=分界 (x=0)
- **共识预测图**: 所有历史预测叠加, 均值线 + ±1σ 置信带, 当前价参考线

## 7. 回测验证

### 7.1 回测逻辑

对日期范围内每一天：
1. 前 N 天数据生成模板
2. 在模板开始前的历史中搜索相似窗口
3. 取 Top K 个匹配的后续收益率均值作为**预测** (从窗口结束**后一天**起算, 与形态匹配一致)
4. 与实际后续收益率比较方向: 同正或同负 = **命中**

### 7.2 快速模式

Pearson 矩阵乘法预计算全部相关系数, 回测降为 O(n):

```
全部窗口 z-score → 堆成矩阵 → Wz @ Wz.T / (n-1) → 相关系数矩阵
逐日 O(1) 查表
```

DTW/Pearson+DTW 模式下也使用 Pearson 矩阵做初筛。

### 7.3 信号段去重

连续同方向预测合并为一个信号段, 消除相邻模板重叠 (>90%) 导致的虚高命中率。

**去重规则**: 段内超过半数交易日正确 → 该段命中。

**报表指标**:
| 原始 | 去重叠 |
|------|--------|
| 回测天数 | 信号段数 |
| 命中次数 | 命中段数 |
| 方向命中率 | 去重叠命中率 (含 delta 对比) |
| 日均匹配数 | 段均天数 |

### 7.4 自动调参

搜索 240 种参数组合, 按去重叠信号段命中率排名, 输出 Top 15:

| 参数 | 搜索值 |
|------|--------|
| 窗口 | 5, 10, 20, 30 |
| 预测天数 | 3, 5, 10, 15 |
| 相似度阈值 | 0.7, 0.8, 0.85, 0.9, 0.95 |
| TopK | 1, 3, 5 |

三种匹配算法均支持, 结果与手动回测完全一致。

### 7.5 时间分布

| 图表 | 内容 |
|------|------|
| 时间线 | 绿/红方块 = 命中/未命中, 悬停看详情 |
| 逐月命中率 | 柱状图 + 命中率折线 + 50% 基线 |
| 30 日滚动命中率 | 识别策略持续有效/失效时段 |

## 8. 数据验证

`python3 validate.py <股票代码>` 自动化 8 大类数据校验：

- **时效性**: 日线 / 估值 / 财报最新日期
- **日线完整性**: OHLC 大小关系、价格非负、涨跌幅计算
- **估值交叉**: PE/PB 合理范围, PB×BPS 与收盘价偏差
- **财务逻辑**: 毛利率 ≥ 净利率, 流动比率 ≥ 速动比率, 资产负债率恒等式
- **技术指标**: RSI [0,100], 布林带顺序, MACD, Beta/波动率/回撤合理范围
- **因子完整性**: 关键因子是否全部存在
- **数据量**: 交易日数是否匹配时间跨度
- **价格修正**: 修正因子连贯性

## 10. 已知限制

1. **数据源**: 新浪/百度/同花顺免费 API, 稳定性受网络影响
2. **估值频率**: 百度 PE/PB 双周频, 插值为日频精度有限
3. **财报滞后**: 季报发布后 1-2 个月才入库
4. **指数数据**: 新浪指数可能滞后一年, Beta 近期偏差
5. **DTW 速度**: 即使 JIT+LB_Keogh, 仍比 Pearson 慢约 10x
6. **形态匹配**: 短期 (2-3天) 被噪声主导, 中周期 (5-10天) 效果更显著
7. **单市场**: 仅支持 A 股, 不支持港股/美股

## 12. 学术定位与迭代路线

### 12.1 当前定位

**工程实现扎实的 k-NN 形态匹配系统** (非参数时间序列预测经典路线)。

| 维度 | 评价 |
|------|------|
| 匹配算法 | Pearson + DTW + 混合, 覆盖了该领域两大标准度量 |
| 加速 | Numba JIT + LB_Keogh + 矩阵乘法, 工程层面较完善 |
| 回测严谨性 | 信号段去重消除模板重叠虚高, 自动调参与手动一致 |
| 因子体系 | 65+ 因子覆盖技术面+基本面+估值, 多因子联合匹配 |
| 差距 | 缺特征学习/自适应权重/非线性降维, 距前沿差两代 |
