# 股票多因子分析系统 — 技术文档

## 1. 项目概述

一个基于 Python 的 A 股多因子分析与历史形态匹配系统。通过免费数据源拉取历史日线和财报数据，计算 60+ 个量化因子，提供交互式 Dashboard 进行可视化分析，支持三种形态匹配算法 (Pearson / DTW / Pearson+DTW)，并提供逐日回测验证和自动参数调优。

**技术栈**: Python 3.9 + Streamlit + Plotly + AkShare + Pandas + NumPy + SciPy + Numba

**在线地址**: https://htma2024-stock-factors-app-pyb4mo.streamlit.app

**仓库**: https://github.com/HTMA2024/stock-factors

## 3. 数据架构

### 3.1 数据源

| 数据层 | 来源 | API | 频率 | 备注 |
|--------|------|-----|------|------|
| 日线行情 | 新浪财经 | `stock_zh_a_daily` | 日频 | OHLCV + 换手率 + 流通股本 |
| 估值数据 | 百度股市通 | `stock_zh_valuation_baidu` | 双周频 → 日频 | PE(TTM)/PE(静)/PB/市现率/总市值 |
| 财务报表 | 同花顺 | `stock_financial_abstract_ths` | 季度 → 日频 | ROE/毛利率/增长率/资产负债率等 |
| 指数行情 | 新浪财经 | `stock_zh_index_daily` | 日频 | 用于计算 Beta |

### 3.2 价格修正机制

新浪前复权价格可能因近期除权除息滞后导致偏差。系统使用独立数据源进行交叉修正：

```
真实股价 = PB(百度) × BPS(同花顺财报)
```

- PB 来自百度双周频数据，线性插值为日频
- BPS 来自同花顺季报，前向填充到日频
- PE/PB/PCF 双周频数据在 factor_engine 中前向填充为日频，消除 NaN 缺口，确保形态匹配有足够的搜索深度

### 3.3 数据缓存

- 所有 API 返回数据缓存到本地 `data/` 目录 (CSV 格式)，按日期范围校验完整性
- Streamlit `@st.cache_data` 缓存因子计算结果 (TTL 1 小时)
- 形态匹配相关系数矩阵独立缓存，权重调整不触发重算

## 6. 历史形态匹配 (核心策略)

### 6.1 策略原理

**历史类比法 (Historical Analogy / Pattern Matching)**

当前一段时间的因子曲线作为"模板"，在历史数据（仅模板开始之前的窗口，排除自身）中搜索形态最相似的窗口，观察相似窗口**之后**的走势，以此作为预测参考。

### 6.2 三种匹配算法

| 算法 | 原理 | 速度 | 适用场景 |
|------|------|------|----------|
| **Pearson** | 线性相关系数，衡量方向一致性 | 最快 (矩阵乘法) | 节奏一致的走势 |
| **DTW** | 动态时间规整，允许 ±2 天伸缩 | 中等 (Numba JIT + LB_Keogh 剪枝) | 同趋势但涨跌快慢不同 |
| **Pearson+DTW** | Pearson 矩阵初筛 (r≥0.3) → DTW 精选 | 中等 | 兼顾速度与灵活性 |

**DTW 加速技术**:
- **Numba JIT**: 编译为机器码，50-100x 加速
- **LB_Keogh 剪枝**: O(n) 下界估计，剪掉 60%+ 不合格窗口，零精度损失

### 6.3 多因子综合评分

```
综合得分 = Σ(因子_i 的相似度 × 权重_i)
```

- 权重可配置，总和自动归一化为 1
- 6 种快捷预设一键切换 (纯价格/量价组合/技术指标/多维度/纯技术/估值+价)

### 6.4 交互配置

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 匹配窗口 | 模板长度 (交易日) | 5/10/20/30/60 |
| 显示前 N 名 | 展示的匹配数量 | 5 |
| 预测展望 | 后续走势显示天数 | 与窗口同 |
| 模板截止日 | 模板窗口的结束日期 | 最新日期 |
| 匹配因子 | 用于匹配的因子 (多选) | close, PE, PB, RSI |
| 因子权重 | 可调滑块, 自动归一化 | 均分 |
| 匹配算法 | Pearson / DTW / Pearson+DTW | Pearson |

### 6.5 可视化设计

- **当前模板图**: 所有选中因子叠加，X 轴为相对天数 (0=截止日)
- **排名分组**: 每个排名一行，每个因子一张小图，并排展示
- **每张小图**:
  - 蓝色实线 = 历史匹配段
  - 蓝色虚线 = 历史预测段
  - 红色实线 = 当前模板
  - 灰色点线 = 匹配/预测分界 (x=0)
- **共识预测图**: 所有历史预测叠加，均值线 + ±1σ 置信带
- **悬停提示**: 鼠标悬停显示实际日期和数值

## 7. 回测验证

### 7.1 回测逻辑

对选定日期范围内的每一天：
1. 生成前 N 天模板
2. 在模板之前的历史中搜索相似窗口（排除模板自身和相邻窗口）
3. 取 Top K 个历史匹配的后续走势均值作为预测
4. 与实际走势比较方向

### 7.2 信号段去重

连续同方向预测合并为一个"信号段"，消除模板重叠导致的虚高命中率。

**去重规则**:
- 相邻两天预测方向相同 → 同属一个信号段
- 段内超过半数交易日正确 → 该段命中

报表同时展示原始命中率和去重叠命中率。

### 7.3 快速模式

Pearson 算法使用 NumPy 矩阵乘法预计算全量相关系数矩阵，回测从 O(t×s×w) 降为 O(t)，秒级出结果。

```
全部窗口 z-score → 堆成矩阵 → Wz @ Wz.T / (n-1) → 相关系数矩阵
逐日只需矩阵查表 O(1)
```

DTW 模式也使用 Pearson 矩阵做初筛过滤。

### 7.4 自动调参

搜索 240 种参数组合 (窗口×预测天数×阈值×TopK)，按去重叠信号段命中率排名输出 Top 15。

| 参数 | 搜索范围 |
|------|----------|
| 窗口 | 5, 10, 20, 30 |
| 预测天数 | 3, 5, 10, 15 |
| 相似度阈值 | 0.7, 0.8, 0.85, 0.9, 0.95 |
| TopK | 1, 3, 5 |

三种匹配算法均支持自动调参，结果与手动回测完全一致。

### 7.5 时间分布

回测结果附带三张时间分布图：
- **时间线**: 每个方块 = 一次预测 (绿命中/红未命中)
- **逐月命中率**: 柱状图 + 命中率折线 + 50% 基线
- **30 日滚动命中率**: 识别策略持续有效时段

## 8. 数据验证

运行 `python3 validate.py <股票代码>` 进行自动化数据校验：

- **时效性**: 检查日线/估值/财报最新日期
- **日线数据**: OHLC 大小关系、价格非负、涨跌幅计算
- **估值交叉**: PE/PB 范围、PB×BPS ≈ 价格
- **财务逻辑**: 毛利率 ≥ 净利率、流动比率 ≥ 速动比率等
- **技术指标**: RSI 范围、布林带顺序、MACD 合理性、Beta/波动率/最大回撤非负

## 10. 已知限制

1. **数据源依赖**: 新浪/百度/同花顺免费 API，稳定性受网络环境影响
2. **PE/PB 频率**: 百度估值数据为双周频，前向填充后精度有限
3. **财报滞后**: 季报发布有约 1-2 个月延迟
4. **Beta 数据**: 新浪指数数据可能滞后 1 年
5. **DTW 速度**: 即使有 JIT + LB_Keogh，DTW 仍比 Pearson 慢一个数量级
6. **形态匹配局限**: 历史匹配不代表未来必然重复；短期 (2-3 天) 预测受噪声主导，中周期 (5-10 天) 效果更显著
